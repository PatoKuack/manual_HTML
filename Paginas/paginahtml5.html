Contenido embebido

Contenido que no es parte del documento html sino que se encuentra en una fuente externa y se inserta en el documento html.
Existen dos tipos de imágenes:

Mapa de bits: conjunto de pixeles que unidos forman una imagen
.jpg .jpeg .png (los png pesan mucho)
.gif (animaciones)
.webp (pesa poco y soporta transparencia pero no es muy compatible con todos los navegadores)
Otro inconveniente es que se tiene que modificar escala de la imagen para ser compatible con cada dispositivo
Se puede usar la página caniuse para verificar las imágenes compatibles con los navegadores

Vectoriales: es código matemático, éstas se pueden escalar (aumentar tamaño) sin perder resolución.
.svg, pesa poco, tiene buena resolución y se renderiza (es compatible) en cada dispositivo.

La etiqueta para agregar imágenes es img, es una etiqueta sin finalización o self closing.
Con el atributo obligatorio src (de source) se coloca el origen o fuente de esta imagen (ejemplo: src="ruta/nombre.jpg").
Se puede colocar el atributo alt="pequeñísima descripción de la imagen" para que, si por alguna razón, el navegador no carga la imagen, aparece en su lugar la descripción.
además de que las personas con discapacidad utilizan programas lectores los cuales leen estas descripciones y ayudan a entender mejor la página,
también sirve para que la imagen aparezca en el navegador de google.
Con los atributos width="#" height="#" (ancho y alto) se modifica el ancho y alto de la imagen.
Las imágenes .svg sólo se pueden modificar en uno de estos dos atributos y se acoplan automáticamente para verse uniformes,
en cambio en las demás imágenes es opcional poner solo uno de los dos atributos o los dos para modificar su proporción.
(aunque con CSS se puede omitir estos dos atributos en HTML).
Se pude usar el atributo title="descripción" para que al sobrepasar el cursor en la imagen, se visualice un cuadro de texto con la descripción colocada en title.

En cada dispositivo las resolución de videos o imágenes es diferente, en los celulares las imágenes de 1800 pixeles se ven muy nítidas por tener disponibles en la pantalla 300 pixeles.
devicePixelRadio es la relación con los pixele reales originales de fabricante que tiene un imagen contra la cantidad de pixeles que se tiene disponible para mostrar el contenido.
Pixeles de viewport: pixeles disponibles para mostrar contenido.
Pixeles reales: pixeles físicos implementados por el fabricante en la pantalla.
devicePixelRadio = PixelesReales/PixelesViewport
Entre más Pixeles reales que Pixeles de viewport haya se verá más nítida la imagen.

Con el atributo srcset se pueden colocar varias imágenes determinando el devicePixelRadio para que el navegador escoja la imagen adecuada para el dispositivo.
ejemplo: img srcset="img/pegasuschico.png 2x, img/pegasusgrande.png x4" src="img/pegasusdefault.png">
En este ejemplo la imagen cambiará dependiendo de cuanto devicePixelRadio soporta la pantalla del dispositivo, x2, x4 o default.
En algunos navegadores como Chrom se guarda esta configuración en la memoria cache al cargar la página por primera vez y no se modificará
la especificación del "srcset" aunque se modifique el código. Aunque en otros navegadores como Firefox no se guarda esta modificación en la memoria cache.


Con la etiqueta picture se pueden utilizar características como la etiqueta "img" pero con más opciones (solo puede haber una etiqueta picture).
Dentro de esta etiqueta se utiliza la etiqueta source para colocar condiciones que se deben cumplir, al final se coloca la etiqueta "img" por si no se cumple ninguna condición "source".
cuando alguna condición source se cumple, no se tomarán en cuenta los source consecutivos.
Dentro de la etiqueta "source" se coloca el atributo srcset=" " colocando entre las comillas la ruta de una imagen.
Dentro de la etiqueta "img" se coloca el atributo src=" " para indicar dentro de las comillas la ruta de la imagen
Las condiciones de los "source" se colocan después de el atributo que indica la ruta de la imagen con el atributo media=" "
Se puede colocar entre las comillas (min-width: 800px) para indicar en este caso que el ancho mínimo para que aparezca la imagen sea de 800 pixeles (si el ancho de pantalla es de más de 800 pixeles aparece la imagen).
El código se lee de arriba para abajo, así que es importante colocar las condiciones de más grande a más pequeños pixeles.
(ejemplo: source srcset="..." media="...800..."
source srcset="..." media="...400..."
img src="...svg">)
Se puede colocar el atributo type=" " en vez de "media" para indicar entre las comillas el tipo de imagen que se colocará y así el navegador si es que soporta la imagen se adaptará al ancho de la pantalla
(ejemplo: type="image/webp">)
Se podría colocar otro "source" con otro type si es que un navegador (por ejemplo safari que no soporta .webp) no soporta el tipo de imagen
Entonces el condicional se saltaría al segundo "type" (ejemplo: type="image/jpg">)
