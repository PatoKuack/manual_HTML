Formularios

El formulario es la parte del sitio web en el que el usuario puede interactuar
escribiendo texto o activando una acción, después se puede procesar la información y se dará una respuesta.
Se crea con la etiqueta form con su inicialización y finalización.
Dentro de la etiqueta de inicialización se colocan atributos como por ejemplo:
El atributo action recibe como valor una URL que se encarga de procesar los datos que contenga el formulario, si no se coloca un valor,
se recarga la página actual y si se coloca un '#' no se recarga la página y hace el efecto de enviar el formulario (ejemplo: action="datos.php").
Estos datos son mandados al servidor el cual evalúa que sean seguros, los procesa y valida.
El atributo method complementa el metodo "action" y permite decidir por que metodo se puede enviar la información (POST o GET) (ejemplo: method="GET").
^ Este último atributo es de una programación más avanzada ^
El atributo autocomplete="on|off" permite autocompletar el formulario con valores guardados (ejemplo: form autocomplete="on">)

Dentro del formulario se colocan textos y campos donde se ingresan valores,
los campos se crean por medio de la etiqueta input (entradas) la cual solo se inicializa.
Se aconseja separar los input encerrando cada uno en etiquetas div
Dentro de esta etiqueta "input" se pueden colocar atributos para cada campo, por ejemplo:

El atributo type diferencia el tipo de entrada que se desea (ejemplo: type="text").
El atributo "type" puede tener muchos valores:
un valor text esta por default (puede colocarse o no) y sirve para colocar un texto,
un valor number permite colocar números en el campo y con un gadget incluido permite aumentar o disminuir la cantidad,
un valor tel simplemente sirve para colocar un número telefónico
un valor date permite colocar una fecha y con un gadget incluido permite seleccionar un día del calendario
un valor password impide ver alguna contraseña que se ingrese aunque no es del todo seguro (es necesario más validaciones de seguridad para que sea seguro),
un valor e-mail valida si el texto ingresado es un posible mail o no,
un valor checkbox crea casillas cuadradas para marcarlas como opciones, permite marcar más de las casillas vinculadas,
un valor radio crea casillas redondas para marcarlas como opción, permite marcar solo una de las casillas vinculadas,
se debe agregar el atributo "name" en estos dos valores de casillas usando el mismo nombre en las casillas vinculadas,
En estos dos atributos (checkbox y radio) se puede agregar el atributo checked para que la casilla esté seleccionada por default.
un valor button crea un botón y con ayuda de un atributo extra "value=" en el mismo input, coloca un texto dentro del botón,
éste atributo realiza una acción con javaScript (ejemplo: type="button" value="Guardar"),
un valor submit crea un botón con un texto por defecto (Submit) que se puede modificar con el atributo "value=",
sirve para crear un botón que manda los datos del formulario a algún lugar especificado en la etiqueta "form",
un valor color permite seleccionar un color por medio de una casilla de colores desplegables,
un valor range permite seleccionar un rango por medio de una barra movible,
con los atributos extras min=" " y max=" " se puede modificar el deslizamiento de la barra,
con el atributo extra step="2" se puede hacer que el deslizamiento de las paradas de la barra sean de 2 en 2 en este caso,
con el atributo "value" en este input se puede inicializar el rango en un valor especifico.

Aparte del atributo type, se pueden utilizan otros como el atributo contenteditable que permite editar el texto directamente
por el usuario en la página web. Este atributo puede utilizarse en cualquier elemento como un <p contenteditable>
El atributo value coloca un texto por default dentro del campo (ejemplo: value="@hotmail.com").
El atributo placeholder hace lo mismo que el atributo "value", solo que el texto dentro del campo
desaparece al querer escribir en el campo (ejemplo: placeholder="Escribe tu nombre").
(Desventaja) Si nuestra página se traduce a otro idioma, el texto del atributo "placeholder" aparecerán en su idioma original
El atributo size permite modificar el ancho del campo pero eso es mejor manipularlo con CSS (ejemplo size="10").
El atributo maxlength limita la cantidad de caracteres que se pueden ingresar al campo (ejemplo maxlength="5").
El atributo name="" se debe usar en cada elemento de los formularios porque establece al servidor
que valor estamos pidiendo, es recomendable que se nombren los campos en inglés
(ejemplo: name="email" es como decir en el programa que en éste caso el input tiene el nombre de "email").
El atributo "name" es importante al procesar los datos.
El atributo id crea un identificador para el elemento donde de usa, esto se usa junto con otras etiquetas (ejemplo: id="nombre").
También se puede usar el atributo class para usarlos con JavaScript (ejemplo: clas="class-name").
El atributo required sirve para indicar que un campo debe ser llenado, es decir que no puede tener un valor nulo, este atributo se coloca solo (ejemplo: <input type="checkbox" required>Acepto términos).
El atributo readonly niega la modificación de un campo por parte del usuario, este atributo se coloca solo (ejemplo: <input type="text" readonly>).
El atributo disable desactiva un campo, algunos navegadores aplican "readonly" a los campos con "disable", este atributo se coloca solo (ejemplo: <input type="text" disable>).
El atributo pattern=" " sirve para las expresiones regulares que es un tema muy extenso y sirven en este caso para especificar la validación de un dato introducido como que lleve .com o que contengan determinado numero de letras y determinado numero de numeros.

La etiqueta textarea con su inicialización y finalización, coloca un campo grande para escribir un texto mas amplio.
Se pueden especificar sus detalles agregando los atributos name="", id="", cols="", rows="" o placeholder=""
"cols" y "rows" definen cuantas columnas y filas puede contener el cuadro de texto aunque eso se puede modificar con CSS.

Los textos de los formularios que están afuera de los campos pueden ser creados dentro de etiquetas label
La etiqueta label con su inicialización y finalización nos permite vincular texto a un "input".
Dentro de ésta etiqueta se coloca el atributo for colocándole de valor el "id" del "input",
así al dar clic en el texto se podrá escribir en el campo (ejemplo: for="id del input").
También se puede colocar un "input" dentro de la etiqueta "label" para vincularlos sin necesidad de un "id" ni "for".

La etiqueta select con su inicialización y su finalización nos permite hacer listas desplegables.
Dentro de esta etiqueta, las etiquetas option con su inicialización y finalización permiten colocar las opciones de la lista (un option por opción)
colocándoles un atributo "value" (ejemplo: option value="Nombre del campo">Nombre visible).
También se le puede anexar el atributo selected para que la opción en ella esté seleccionada por default.
Con la etiqueta optgroup con su inicialización y finalización permite agrupar las opciones (ejemplo: optgroup label="aficiones">)

La etiqueta fieldset permite crear listados de opciones, al empezar a escribir en el campo se buscan las opciones posibles de acuerdo a lo que está en el campo
dentro de esta etiqueta se agrega la etiqueta "input" de tipo type="list"
con el atributo list="id del datalist" se vincula la lista de opciones,
dentro de este input se agrega la etiqueta datalist que es donde van a ir las opciones
es importante agregarle un atributo id="" para vincularlo al "imput" dentro de esta etiqueta "datalist" van las opciones con la etiqueta "option"
^ La lista de opciones (fieldset) actualmente (octubre 2020) no está bien habilitada en FIREFOX ^
*
También, la etiqueta fieldset con su inicialización y su finalización agrupa los campos del formulario encerrándolos en un margen.
La etiqueta legend con su inicialización y su finalización coloca una titulación de la agrupación (ejemplo: legend>Datos personales).

La etiqueta button con su inicialización y finalización crea un botón, dentro de esta etiqueta se coloca el nombre visual del botón
y por default éste elemento tiene como atributo type="submit", con el atributo onclick="funcin()" se puede agregar una URL
o una función de JavaScript para que se redireccione o ejecute respectivamente cuando se le da clic al botón
con la etiqueta formaction="URL" se especifica donde mandar la información del formulario,
y con el atributo form="form_id" se especifica el formulario del que se quiere mandar los datos
(en caso de que haya más de un formulario en la página) con ayuda del id de uno de los formularios de la página.

Formularios con IFrames

También se pueden colocar formularios con la ayuda de IFrames, ejemplo:
Se puede obtner un formulario facilmente con la página Facebook for developers, nos dirigimos a la pestaña Docs o Documentos y de ahí al bloque de Integraciones sociales, en este bloque buscamos y accedemos a la opción Social plugins y de ahí se selecciona la opción que nos interese, en este caso será la pestaña Plugin de páginas, en esa opción se diseña el IFrame y una vez diseñada se presiona el botón Obtener código y se copia el código del IFrame que se encuentra en la segunda pestaña de la ventana emergente. Se pega el IFrame en donde se desee del código HTML, se suele colocar dentro de una etiqueta div y lo q hará es pedir al usuario escribir un texto y mandarnoslo a nuestro inbox de Facebbok.
