Enlaces

Enlaces : son aquellos textos subrayados color azul que permite redirigir a otros sitios web al darles clic.
La etiqueta a (de anchor = ancla) con su inicialización y finalización sirve para enlazar un texto y abrir el enlace en la pestaña actual.
Con el atributo href=" proyecto.extención " se indica el link del enlace. La palabra que va a servir de enlace al momento de darle clic va dentro de la etiqueta.
El atributo title es omisible del código, éste crea un pequeño recuadro al poner el apuntador del mouse sobre la palabra enlazada.
(ejemplo: a href="URL" title="cuadro de texto que aparece al sobrepasar el mouse"> PALABRA <).
Existen referencias especificas para los enlaces, por ejemplo, href="mailto:correo@mail.com" abre el correo del usuario que se tenga instalado en la máquina, creando automáticamente un correo colocando como destinatario el correo especificado (correo@mail.com).
Para abrir el enlace en una pestaña nueva se usa el atributo target="_blank", esté atributo es útil cuando se enlaza contenido ajeno a la de la página o a la del autor de la página
(ejemplo: a target="_blank" href=...).
Por cada enlace que redirecciona a una página, se le aumenta SEO (Search Engine Optimization o posicionamiento en buscadores)
El atributo rel="nofollow" evita que los enlaces que haya en la página no transmitan SEO, esto es útil en páginas donde usuarios pueden comentar y colocar enlaces como Facebook.
El atributo rel=noreference" evita que se indique al destino el origen del link, es decir si una página enlaza a otra, con éste comando la otra página no podrá saber de dónde lo enlazaron.
El atributo rel="noopener" evita la incrustación de código malicioso al abrir enlaces en nuevas pestañas, lo cual evita hakeos.
El atributo download descarga el recurso en vez de abrirlo en el navegador, el archivo tiene que encontrarse en mi servidor para poder mandarlo descargar con éste atributo.
(ejemplo: a href="pegasus.png" download> descargar a Pegasus)
Las rutas relativas son las que direccionan a una carpeta de donde extrae un archivo (png, doc, txt, etc.).
Para ir a rutas relativas que están afuera de la carpeta raíz se colocan dos puntos antes de la diagonal (ejemplo: href="../img/pegasus.png")
Para ir a la carpeta raíz directamente en rutas relativas se coloca una diagonal en la ruta de href (ejemplo: href="/") generalmente en la carpeta raíz se encuentra el index que es el archivo que contiene un menú con las rutas a la mayoría de los enlaces.
Las rutas absolutas son las que nunca cambiaran, las URL (sin olvidar el protocolo como http://) que nos direcciona a otra página ajena a nuestro proyecto.
Enlaces hash.
Se coloca gato (#) y el id del título.
(ejemplo de enlace: a href="#cap1">cap 1)
(ejemplo de id a un título h2: h2 id="cap1"> Capitulo 1)
Enlaces hash a una página externa:
Se coloca la URL y una gato seguido del nombre especifico del título de la página externa.
(ejemplo: href="https://ed.team#comunidad") en este ejemplo se dirigiría el enlace al sitio web de edTeam al título o sección "comunidad"

Links

Los links se ponen entre los head generalmente
Se puede mandar a llamar un estilo CSS mediante la etiqueta link rel="stylesheet" href="ruta del archivo.css"
encerrada entre <> sin finalización. "stylesheet" indica que será de estilo hoja en cascada.
Se puede enlazar archivos de tipo JavaScript con el comando script src="ruta del archivo.extensión"
encerrada entre <> con su seguida finalización /script.
En la sección 10 se encuentra un poco de información adicional.
Para agregar un icono a la página se escribe el comando
link rel="shortcut icon" type="image/x-icon" href="ruta del archivo.ico" encerrada entre < >,
rel indica que será un icono y type indica el tipo de imagen que será.

Se pueden escribir etiquetas en HTML sin activar el código con unicode, ejemplo:
    &#60;  = <
    &#62;  = >
    &#47;  = /
    &#160; = Espacio simple
    &nbsp; = Espacio simple
    &ensp; = Espacio doble
    &emsp; = Espacio cuadruple
    &nbsp;&nbsp;&nbsp;&nbsp; = Sangria
